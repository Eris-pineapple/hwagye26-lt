# 🏟️ 26화계LT — 팀 대항 파티 게임 (게임 올림픽)

각자 휴대폰으로 접속하는 팀 대항 파티 게임. 관리자 화면(노트북→TV 미러링)이 전체 공유 화면 역할을 하고, 플레이어 화면(폰)은 입력 장치 역할을 합니다.

## 화면

| 파일 | 용도 |
|---|---|
| `index.html` | **플레이어 화면** — 폰으로 접속, 닉네임+PIN 로그인 |
| `admin.html` | **관리자 화면** — 노트북으로 열어 TV에 미러링. 게임 제어 + 점수판 + 연출 + 음성 |

## 게임 종목 (게임 올림픽 컨셉)

1. **🚨 긴급 출동! 색상 수사대** — 목표 색과 비슷한 물건을 60초 내 촬영, Claude가 유사도 0~1000점 감식. 팀 평균제.
2. **🚀 정각에 흔들어! (발사 관제센터)** — T-0 정각에 폰을 흔들기(DeviceMotion). `S(δ)=1000·e^(-0.003|δ|)/(1+0.0001δ²)`. 팀 합산제 + **인원수 비율 보정** (적은 팀 기준 환산).
3. **🖼️ 고장난 전화기 미술관 (그림 릴레이)** — 제시어→그림→따라 그림→...→추측. 유사도 70% + 정답 30%, **릴레이 단계 수 비율 보정**. 종료 후 변천사 슬라이드쇼.
4. **🦑 스피드 OX 서바이벌** — Claude 즉석 출제, 틀리면 탈락. 생존 비율 ×1000점 (비율 보정 내장).

종료 후 **🏅 시상식**: Claude가 전체 게임 기록을 보고 MVP상/굴욕상 등을 즉석 생성, 한 명씩 두구두구 공개.

## 배포 (Vercel — 권장)

```bash
npm i -g vercel
cd hwagye26-lt
vercel          # 또는 vercel --prod
```

**환경변수 설정 (Vercel 대시보드 → Settings → Environment Variables):**

| 변수 | 용도 |
|---|---|
| `ANTHROPIC_API_KEY` | Claude 채점/생성 (`/api/claude` 프록시) |
| `GEMINI_API_KEY` | Gemini TTS 음성 (`/api/tts` 프록시) |

키는 서버리스 함수에서만 사용되어 클라이언트에 노출되지 않습니다.

## Firebase 설정 (필수 1회)

Realtime Database: `https://hwagye26-default-rtdb.asia-southeast1.firebasedatabase.app/`

콘솔 → Realtime Database → 규칙에서 읽기/쓰기를 허용해야 합니다 (파티용 간이 설정):

```json
{
  "rules": { ".read": true, ".write": true }
}
```

> ⚠️ 누구나 읽고 쓸 수 있는 규칙입니다. 행사 기간에만 열고, 끝나면 닫는 것을 권장합니다.

## 로컬 실행

### 방법 A — `vercel dev` (서버리스 포함, 권장)
```bash
vercel dev
# .env 파일에 ANTHROPIC_API_KEY, GEMINI_API_KEY 작성
```

### 방법 B — 정적 서버 (AI 기능은 임시 키 입력 필요)
```bash
python -m http.server 8000
# 또는 npx serve
```
서버리스가 없으므로 관리자 화면 ⚙️ 설정에서 API 키를 임시 입력하면 브라우저에서 직접 호출합니다(localStorage 저장, 로컬 테스트 전용).

> 📱 폰 카메라/모션 센서는 **HTTPS 또는 localhost**에서만 동작합니다. 실제 행사에서는 Vercel 배포 주소를 쓰세요.

## 진행 순서 (관리자 가이드)

1. **로비/팀**: 팀 수 설정 → 플레이어 접속 확인 → 팀 배정. 팀별 인원수가 달라도 됩니다(자동 보정).
2. **개회식** 버튼으로 팀 소개 연출 (TTS).
3. **게임 선택**에서 종목 시작. 진행은 대부분 자동(타이머/전원 제출 시 자동 채점).
4. 폰이 꺼진 플레이어는 로비에서 **건너뛰기** 토글 → 채점/대기에서 제외. 그림 릴레이는 팀 보드의 **현재 주자 건너뛰기** 버튼도 사용 가능.
5. 모든 종목 후 **시상식** → AI가 상을 만들고 한 명씩 공개.

## 운영 메모

- **재접속**: 같은 닉네임+PIN으로 다시 로그인하면 팀/점수/진행 중 라운드 모두 복원. 같은 기기는 자동 로그인.
- **AI 모델**: 헤더에서 Haiku(빠름)/Sonnet(정밀) 선택.
- **음성**: 관리자 화면에서만 출력. 🔊 토글로 ON/OFF. 카운트다운 등 즉각 반응은 Web Speech, 결과 발표·시상 멘트는 Gemini TTS(실패 시 Web Speech 자동 대체). TTS는 두구두구 연출 중에 미리 생성됩니다.
- **iOS 센서**: '정각에 흔들어'에서 폰의 "센서 점검" 버튼을 눌러 권한을 허용해야 합니다.
- **사진**: 폰에서 촬영 → 720px JPEG로 압축해 RTDB에 base64 저장 (Storage 불필요).
